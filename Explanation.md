The repository contains four Google Colab-ready Jupyter notebooks (.ipynb) representing the main stages of the Urban Visual Reprogrammability (UVR) workflow.

a.GSV_Download.ipynb — Google Street View Data Collection

Downloads Google Street View (GSV) images within a selected geographic area.
The notebook requires the coordinates defining the bounding rectangle of the study area and automatically samples and retrieves GSV imagery.

b.ObjectDetection_GroundingDINO.ipynb — Object Detection with Grounding DINO

Performs object detection on the downloaded GSV images using the Grounding DINO model.
The notebook identifies and counts urban elements associated with potential visual reprogrammability (e.g., awnings, bus shelters, billboards, fences, umbrellas, etc.).

c.ExcelFileImplementedByMapillary.ipynb — Mapillary Data Integration

Enhances the Excel output generated by the previous notebook by integrating detections obtained from Mapillary.
This step combines the outputs of Grounding DINO and Mapillary into a unified dataset.

d.ExcelFileWithUVRIndex.ipynb — UVR Index Computation

Generates the final Excel dataset containing the Urban Visual Reprogrammability (UVR) Index for each geographic point.
After manually assigning contribution scores to each detected urban element class, the notebook computes the UVR Index using the methodology described in the research project.

The repository also includes the full research paper titled “Reimagining the City as an Editable Canvas_FF_FM”, containing the conceptual framework, methodology, taxonomy, scoring system, and discussion of the Urban Visual Reprogrammability (UVR) research project.
